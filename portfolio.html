Burdell's Car Dealership App

Technologies used: Python, Flask, PostgreSQL, JavaScript, HTML, BootStrap

This was a web app built on the Flask framework as part of the CS6400: Database Systems and Design course at Georgia Tech. In a group of 4, we designed a relational database schema according to the given specifications.

Key functionalities:

  • Ability to read from the database all vehicles that matches the search fields
  • Transactions such as vehicle purchases, sales, and repairs can be verified in the controller and written to the database
  • Providing different views and permissions dynamically based on the user type of the logged in user
  • Aggregate sales, purchases, repairs, and vehicle data to generate various reports

Mak Webapp

(Code Available upon request)

Technologies used: Ruby on Rails, Cucumber, Rspec testing, JavaScript, Bootstrap

This was a web app build on Ruby on Rails as part of the CS169: Software Engineering as a Service course at UC Berkeley. As a group of 6 students, we designed, built, and tested this app based on client specifications. Our client was Mak, a student launched venture trying to connect urban low-income youths with STEM skills training and hands-on projects.

Key Features:

  • Administrative users are able to login and view all companies, students, and projects in the system as well as approve new projects and assign students to them
  • Companies can assign tasks and projects to students who are assigned to them
  • Students are able to communicate directly with group members and companies through the forum
  • Agile software development techniques, such as Test driven development, Behavior driven development, and pair programming, were used throughout the project to facilitate rapid web development

Bear Maps

(Code Available upon request)

Technologies used:Java

As part of the CS61B, Data Structures & Algorithms, course at UC Berkeley, I built the backend logic of an interactive map of Berkeley using Java.

Key Features:

  • Store map tiles in a quadtree data structure to support efficient rastering during map generation
  • Implemented a Trie data structure to support autocomplete during location search
  • The A* Algorithm was implemented using a Minimum Priority Queue to find the shortest route between two nodes
